# SwiftAndroid IMDB Demo вҖ” iOS client

A minimal SwiftUI app that demonstrates consuming the per-screen viewmodels from [SwiftAndroidIMDBSdk](https://github.com/erikg84/SwiftAndroidIMDBSdk) v1.1.0+ via the SPM wrapper repo [`swift-android-idbm-sdk-spm`](https://github.com/erikg84/swift-android-idbm-sdk-spm).

Five screens, five SDK viewmodels, **zero wrapper classes**, **zero shared viewmodels** вҖ” each screen owns exactly one `TMDBContainer.get<Screen>ViewModel()` and renders the result.

## One-time setup

### 1. `~/.netrc` for GitHub Packages auth

The XCFramework lives in GitHub Packages Maven, which requires Basic Auth even for public artifacts. SPM (Xcode 13.3+) reads `~/.netrc` automatically.

```bash
# Create a GitHub PAT with `read:packages` scope at:
#   https://github.com/settings/personal-access-tokens/new
# (Public repos read-only + Account вҶ’ Packages: Read)

cat >> ~/.netrc <<'EOF'
machine maven.pkg.github.com
  login <your-github-username>
  password <your-pat>
EOF
chmod 600 ~/.netrc
```

### 2. TMDB bearer token

Get one at https://www.themoviedb.org/settings/api. Two ways to provide it:

- **Easiest**: drop a single line into `SwiftAndroidImdbDemo/tmdb-token.txt`. The file is `.gitignore`'d. The `TokenLoader` reads it from the bundle.
- **Or**: set `TMDB_BEARER_TOKEN` in the Xcode scheme вҶ’ Run вҶ’ Arguments вҶ’ Environment Variables.

### 3. Generate the Xcode project

The `.xcodeproj` is **not** committed вҖ” it's generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml`.

```bash
brew install xcodegen   # if you don't have it
xcodegen generate
open SwiftAndroidImdbDemo.xcodeproj
```

Then build & run on a simulator (iOS 16+) from Xcode.

## Project structure

```
project.yml                              # XcodeGen spec
SwiftAndroidImdbDemo/
в”ңв”Җв”Җ SwiftAndroidImdbDemoApp.swift        # @main entry вҖ” calls TMDBContainer.getShared().configure(token)
в”ңв”Җв”Җ RootView.swift                       # TabView host
в”ңв”Җв”Җ tmdb-token.txt                       # local secret (gitignored)
в””в”Җв”Җ Screens/
    в”ңв”Җв”Җ HomeScreen.swift                 # TMDBHomeViewModel
    в”ңв”Җв”Җ MoviesScreen.swift               # TMDBMoviesViewModel
    в”ңв”Җв”Җ TVShowsScreen.swift              # TMDBTVShowsViewModel
    в”ңв”Җв”Җ SearchScreen.swift               # TMDBSearchViewModel + task(id:) debounce
    в”ңв”Җв”Җ TrendingScreen.swift             # TMDBTrendingViewModel + day/week segmented control
    в””в”Җв”Җ UiState.swift                    # Generic state holder
```

**No DI framework. No view-model wrappers. No shared state.** Each `View` literally writes:

```swift
private let vm = TMDBContainer.getHomeViewModel()
```

The SDK accessor is the dependency. The screen owns its own state via `@State`, fetches in a `.task { }` block, and renders.

## What each screen demonstrates

| Screen | SDK VM | iOS pattern shown |
|---|---|---|
| Home | `TMDBHomeViewModel` | Simplest case: `private let vm = ...` + `@State` + `.task` |
| Movies | `TMDBMoviesViewModel` | Same with a different return type |
| TV Shows | `TMDBTVShowsViewModel` | Same |
| Search | `TMDBSearchViewModel` | `.searchable` + `.task(id: query)` + `Task.sleep` debounce |
| Trending | `TMDBTrendingViewModel` | `.task(id: window)` to refetch on day/week toggle |

## SDK reference

This app references the **published** SDK via the SPM wrapper repo вҖ” not a local checkout. To bump the SDK version, edit `project.yml`:

```yaml
packages:
  SwiftAndroidSDK:
    url: https://github.com/erikg84/swift-android-idbm-sdk-spm
    from: "1.1.0"
```

Then `xcodegen generate && open SwiftAndroidImdbDemo.xcodeproj`.
